# K线结构相似度分析工具算法重构设计

## 目标

重构 `D:\tdx\K线结构相似度分析工具.html` 的算法模块，修复复权、候选筛选、统计有效性和取消响应问题，同时保持单 HTML 文件交付、现有页面布局和用户操作方式不变。

## 约束

- 页面继续支持直接双击打开，不引入构建步骤或运行时第三方依赖。
- 通达信行情和权息数据只在本机读取，不发起网络请求。
- 保留目录选择、个股分析、两种匹配模式、图表、排序与 CSV 导出功能。
- 测试和设计文件放在 `D:\AIAPP`，不污染通达信安装目录。
- 修改前在 `D:\AIAPP\backups` 保存带时间标识的原文件副本。
- `D:\tdx` 不是 Git 仓库，本次不执行 Git 提交，以备份和自动化回归测试作为恢复保障。

## 模块设计

Worker 源码仍嵌入 HTML，但内部按以下边界组织，并通过一个仅在测试环境启用的导出对象暴露纯函数：

1. 数据解析：验证 `.day` 记录长度、日期、OHLC、成交量和成交额。
2. 权息处理：解密 `gbbq`，产生明确的 `valid/raw/error` 状态，并应用价格和成交量复权。
3. 序列计算：负责标准化、重采样、余弦相似度、DTW、ZigZag、均线、波动率和回撤。
4. 特征提取：将参考窗口和候选窗口转成统一特征对象。
5. 候选选择：每只股票先建立局部候选池，再合并成全市场粗筛池，最后执行 DTW 精排。
6. 去重与统计：按股票和窗口重叠度聚类，生成展示结果及独立统计样本。
7. 任务控制：管理缓存版本、进度、取消信号和 Worker 消息。

## 复权规则

- 只有 `gbbq` 解密成功且存在有效权息记录时，全局权息状态才为 `valid`。
- 文件存在但解密为空、格式错误或记录无效时，状态为 `error`，所有证券明确标记为未复权。
- 无 `gbbq` 文件时状态为 `raw`。
- 现金分红、送股和配股沿用通达信除权参考价公式调整历史价格。
- 仅送股和配股改变股本：除权日前历史成交量按股本扩张比例反向调整；纯现金分红不调整成交量。
- 复权状态、有效事件数和异常原因随序列结果返回，UI 不再从文件时间戳推断状态。

## 匹配与候选选择

- 历史模式继续使用32点余弦进行粗筛、完整特征评分和48点 DTW 精排。
- 每只股票最多保留固定数量的局部粗筛候选，局部候选完成重叠去重后再进入全市场候选池。
- 全市场候选按基础综合分裁剪，避免单只股票的大量窗口挤占名额。
- 最终排序仍尊重用户权重和每股展示上限。
- 同期模式继续按共同交易日期对齐；缺失比例超过阈值时剔除，否则只在共同日期上构造全部特征，避免使用错位的连续切片。

## 防前视与回测统计

- 开启防前视时，历史窗口的结束日期必须严格早于参考窗口开始日期。
- 未来收益必须从匹配窗口结束后的第一个交易日之后计算，不允许访问参考窗口选择时不可见的数据来参与评分。
- 展示候选按“同股票且窗口重叠超过70%”聚类；回测汇总每个聚类只取综合分最高的代表窗口。
- 页面同时显示展示样本数和统计有效样本数。
- 胜率95%区间使用 Wilson 区间，不再使用小样本表现较差的正态近似。
- 页面继续提示幸存者偏差，并新增窗口重叠、样本相关和数据挖掘偏差提示。

## 交互改进

- 快捷日期按钮在参考股票数据已加载时按实际交易日回推；尚未加载时采用自然日近似并明确提示。
- Worker 在股票循环和滑窗循环中定期让出执行权并检查取消状态。
- 权息读取失败时显示具体原因，不再同时出现“读取失败”和“已前复权”的矛盾状态。
- CSV 增加权息状态、统计样本标记和算法版本字段。

## 缓存

- 提升算法缓存版本。
- 缓存键包含行情文件大小、修改时间、权息文件指纹和算法版本。
- 权息状态为 `error` 时不写入伪复权缓存。
- 旧版本缓存自然失效，无需破坏性清空整个 IndexedDB。

## 回归测试

使用 Node 内置 `node:test` 与 `assert`，不增加第三方依赖。测试从 HTML 提取 Worker 源码，在隔离上下文中加载纯算法接口。

必须覆盖：

1. `.day` 正常记录、截断记录和非法价格。
2. 无权息文件、有效解密、空解密和解密异常的状态。
3. 纯现金分红、送股、配股及组合事件的价格复权。
4. 送股和配股的成交量反向复权，以及纯现金分红不改变成交量。
5. 常量序列、短序列、完全相同与完全相反序列的余弦、DTW、ZigZag行为。
6. 防前视边界必须严格早于参考开始日期。
7. 每股局部候选池、跨股票合并和70%重叠去重。
8. Wilson 区间在零样本、小样本和大样本下的结果。
9. 同期模式缺失交易日的共同日期对齐。
10. 缓存版本或权息指纹变化后缓存失效。
11. Worker 消息成功、错误和取消路径。

测试必须遵循红—绿循环：先运行新测试并确认因缺少目标行为而失败，再实施最小修复并确认通过。最终还需验证 Worker 与主线程脚本语法、全部回归测试以及 HTML 关键元素完整性。

## 验收标准

- 权息为空或失败时不再标记为前复权。
- 送转和配股事件前的成交量得到正确调整。
- 粗筛结果包含更均衡的股票覆盖，且仍遵守最终每股上限。
- 回测汇总使用去重后的有效样本与 Wilson 区间。
- 实际交易日快捷区间和取消操作按设计工作。
- 所有新增回归测试通过，Worker 和主页面脚本语法检查通过。
- 修改后的文件仍可作为一个独立 HTML 页面打开并读取本地通达信目录。
