# 近期相似区间模式设计

## 目标

为 `D:\tdx\K线结构相似度分析工具.html` 增加第三种匹配模式“近期相似区间”，在每只证券自身最近可调数量的有效交易日内，搜索与目标区间长度相同的相似窗口。

## 用户界面

- 在现有“历史滑窗搜索”和“同期联动”之后增加“近期相似区间”。
- 增加数字输入“近期范围”，允许输入15至5000个交易日。
- 默认值为自动：实际使用目标区间交易日数量 `L`。界面显示“自动＝目标 L 日”。
- 用户输入数值后使用该值，但有效范围不得小于目标窗口长度；若小于 `L`，运行时自动提升为 `L` 并提示。
- 近期模式继续使用滑窗步长、每股上限、市场范围和评分权重。
- 近期模式不使用防前视。选中近期模式时禁用防前视复选框，并显示“近期模式按每只证券最新数据向前搜索”。切回历史模式时恢复此前的防前视选择。

## 搜索规则

- 参考窗口长度为 `L` 个交易日。
- 对每只证券取最后 `N` 个有效交易日，`N` 为近期范围；自动模式下 `N=L`。
- 候选窗口起点范围为 `max(0, bars.length-N)` 至 `bars.length-L`，按现有滑窗步长移动。
- 为确保最新窗口不会因步长遗漏，无论步长如何，始终额外评估以该证券最新交易日结束的窗口。
- 当证券总数据少于 `L` 时跳过。
- 目标证券也允许参与近期比较，但与参考区间重叠超过70%的窗口必须排除，避免目标窗口与自身得到无意义满分。
- 近期模式沿用现有粗筛、特征评分、DTW精排、每股候选上限和重叠去重逻辑。

## 日期语义

- 近期范围按每只证券自身有效交易记录计数，不按自然日截断。
- 停牌不会制造空白交易日；不同证券可以具有不同的近期起始日期和最新日期。
- 结果表继续显示每个候选窗口的实际开始和结束日期。

## 统计与展示

- 结果标题显示“近期相似区间”、目标窗口长度和实际近期范围。
- 最新窗口通常没有足够的后续5/10/20/60日数据；对应字段显示“—”，统计卡仅使用确实存在后续数据的窗口。
- 当某个周期没有有效未来样本时显示“样本不足”，不补零、不推测。
- 风险提示说明近期模式主要用于寻找当前或近期形态，不等价于历史回测。
- CSV 增加模式及近期范围字段。

## Worker 协议

主线程向 Worker 的匹配配置增加：

- `mode: "recent"`
- `recentBars: null | number`，`null` 表示自动使用 `L`

Worker 返回的 `meta` 增加 `recentBars`，值为本次实际使用的范围。

## 测试

采用现有 Node 回归测试框架，先写失败测试，再实现功能。必须覆盖：

1. 自动范围使用 `L`。
2. 自定义范围小于 `L` 时提升为 `L`。
3. 自定义范围大于 `L` 时生成多个滑窗。
4. 滑窗遵守步长且始终包含最新结束窗口。
5. 证券数据不足 `L` 时无候选。
6. 近期范围按每只证券自身交易记录计数。
7. 目标证券与参考区间重叠超过70%时排除。
8. 近期模式忽略防前视设置。
9. 没有未来数据时收益字段为 `null`，统计显示样本不足。
10. 模式切换时防前视控件状态正确。
11. CSV 和结果元数据包含近期范围。

## 验收标准

- 第三种模式可选择并能成功完成匹配。
- 未填写近期范围时，默认搜索每只证券最后 `L` 日的一个最新窗口。
- 调大近期范围后能够在该范围内执行滑窗搜索，且不会遗漏最新窗口。
- 结果、统计、CSV和提示文本符合上述规则。
- 原有历史和同期模式的回归测试继续通过。
- 全部脚本语法、关键控件与本地数据策略验证通过。
