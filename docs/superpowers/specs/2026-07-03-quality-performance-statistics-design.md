# K线工具质量、性能与统计改进设计

## 范围

本轮保持“单 HTML 下载即用”和纯本地运行不变，依次改进数据正确性、扫描性能、参数可解释性、统计独立性和源码维护方式。

## 数据层

建立唯一的 32 字节 `.day` 记录解析函数，返回日期、OHLC、成交额和成交量，并统一过滤非法、乱序和重复记录。复权函数生成逐日价格及成交量因子，主线程和 Worker 使用同一解析结果；无法对齐时返回明确原因。扫描异常按阶段和原因聚合，不再静默吞掉。

## 匹配层

同期模式先按目标日期二分截取候选数据，再做共同日期对齐。普通用户使用“稳健、均衡、宽松”预设；高级区允许调整粗筛阈值、粗筛候选数、DTW 候选数和带宽。结果元数据及 CSV 记录实际参数，并展示各阶段淘汰计数。

## 统计层

展示样本与统计样本分离。统计样本先做同证券重叠去重，再按匹配结束日期的 5 个交易日近邻聚类，每个时段簇对收益取均值，使用簇级 bootstrap 95% 区间。近期模式不把未来数据不足的窗口纳入相应期限统计；界面同时显示原始有效样本数和独立时段数。

## 工程层

算法源码迁入 `src/` 中的可测试模块，并由构建脚本生成 Worker 文本及最终单 HTML；根 HTML 和 `public/index.html` 均由同一构建输入生成。隐私测试扩展到 Beacon、外部资源 URL、动态脚本和表单提交；兼容性检查对缺失 API 给出明确降级提示。

## 验收

- 非法日线记录在主线程与 Worker 得到一致处理。
- 复权对齐失败可见且有测试。
- 同期模式不扫描目标区间外的大段历史。
- 所有高级参数可追溯且三个预设有固定值。
- 统计显示独立时段数和簇级区间。
- 构建可重复生成一致发布物，无网络能力测试覆盖扩大。
