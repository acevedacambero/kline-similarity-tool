# K线相似度工具统计有效性与诊断增强设计

日期：2026-07-03

## 1. 目标与范围

本次增强覆盖以下七项：基准超额收益、配对随机窗口基线、近期样本成熟度、幅度相似度、缓存容量控制、`gbbq` 自检和匹配数量漏斗。

继续遵守现有约束：成品为单HTML文件、只读取通达信本地数据、零网络请求、算法在Web Worker中执行、结果仅用于研究分析而非买卖建议。

不在本次范围内：多Worker并行、ZigZag自适应、DTW评分重构和聚合序列缓存。

## 2. 总体架构

保持 `src/page.template.html`、`src/algorithm.js` 和构建生成物的现有结构。在 `algorithm.js` 内按职责增加五组可独立测试的纯函数：

1. 基准日期对齐与超额收益；
2. 确定性随机窗口采样与配对；
3. 样本成熟度和统计汇总；
4. 幅度相似度；
5. 缓存淘汰与 `gbbq` 有效性诊断。

Worker负责扫描、计算和汇总；主线程只负责识别本地文件、传递配置和渲染结果。

## 3. 本地指数与基准映射

目录扫描除现有证券文件外，还识别以下通达信指数日线：

| 证券范围 | 默认基准 | 通达信键 |
|---|---|---|
| 沪深主板 | 沪深300 | `sh000300` |
| 创业板 | 创业板指 | `sz399006` |
| 科创板 | 科创50 | `sh000688` |
| 北交所 | 北证50 | `bj899050` |
| ETF | 沪深300兜底 | `sh000300` |

指数文件单独保存，不进入普通证券匹配池，也不受ST、新股和板块过滤影响。

如果对应指数缺失，绝对收益照常计算，超额收益记为缺失；界面必须明确显示缺少的基准和受影响样本数，不能静默替代为零收益。

## 4. 超额收益定义

对每个匹配窗口及5、10、20、60周期未来端点：

1. 使用证券实际起止日期；
2. 在基准序列中选择不晚于起止日期的最近交易日；
3. 若起点或终点无法对齐，超额收益为空；
4. 证券和基准均使用对数收益；
5. 展示值为 `exp(证券对数收益 - 基准对数收益) - 1`。

原始绝对收益继续保留。统计面板默认突出超额收益，同时允许用户查看绝对收益，CSV同时导出证券收益、基准收益和超额收益。

## 5. 配对随机窗口基线

### 5.1 候选池

扫描每只证券时，使用固定种子和确定性哈希，从合法窗口中最多抽取8个安慰剂窗口。每条只保存：证券键、板块、周期、起止索引与日期、实现波动率、绝对收益、基准收益、超额收益和样本成熟度。

随机窗口遵守与当前模式相同的数据可用性、上市天数、停牌新鲜度和防前视约束，但不要求通过形态粗筛。

### 5.2 配对规则

每个最终相似窗口在安慰剂池中优先匹配：

1. 相同板块和周期；
2. 不与该相似窗口来自同一证券；
3. 结束日期距离最小；
4. 对数波动率距离最小；
5. 对应未来周期收益有效性一致。

### 5.3 重复统计

使用固定随机种子重复200轮。每轮为每个相似窗口选择一个符合条件的安慰剂窗口，不重复使用同一条安慰剂记录，池不足时允许减少该轮样本并报告实际样本数。

输出：

- 相似样本的超额胜率和中位超额收益；
- 随机基线的胜率和中位收益；
- 两者差值；
- 200轮中相似样本优于随机基线的比例；
- 配对成功率和实际随机样本数。

该结果是描述性安慰剂检验，不宣称为严格的因果推断或已校正全部数据挖掘偏差。

## 6. recent模式样本成熟度

对每个未来周期分别统计：

- 展示样本数；
- 有效收益样本数；
- 有效样本完整率；
- 独立结束时段数；
- 有效样本从窗口结束到证券最后交易日的中位滞后周期；
- 无效样本数。

滞后使用当前日/周/月聚合后的周期数，不混用自然日。有效样本不足时显示“样本不足”，不计算胜率区间。

## 7. 幅度相似度

新增独立子分 `amp`，默认权重10，可设为0关闭。原有权重继续自动归一化。

幅度分由两部分组成：

- 70%：窗口实现波动率比例；
- 30%：窗口收盘价最高与最低对数价格之差的比例。

每部分使用对称分数 `exp(-abs(log(candidate / reference)))`。分母为零或数据无效的窗口使用中性分0.5并记录诊断，不产生无穷值或NaN。

`amp` 不替代现有 `vdd`；后者继续表示波动与回撤综合结构，后续版本可根据实证结果再拆分。

## 8. IndexedDB容量控制

数据库结构升级，股票缓存记录增加估算字节数和访问批次；元数据保存总记录数、估算总字节数和当前访问批次。

规则：

- 最多缓存7000只证券；
- 同时不超过 `navigator.storage.estimate()` 返回配额的20%；
- 配额接口不可用、页面由不支持该接口的本地上下文打开，或估算失败时，只执行7000只上限；
- 每次完整匹配结束后检查一次容量；
- 超限时按最久未使用批量删除，直到降到上限的90%，避免每次匹配反复淘汰；
- 访问时间按批次集中更新，禁止每次缓存命中都启动写事务；
- 版本不匹配的旧记录优先删除。

数据库升级必须兼容当前 `kline_tool_v2`，并避免与页面启动时的旧东方财富缓存清理发生版本竞争。

## 9. gbbq结构与连续性自检

解密结果不再以“至少一条有效记录”作为唯一成功条件。自检包含：

1. 至少解出100条有效事件并覆盖至少30只证券；
2. 证券代码为6位数字；
3. 日期位于1990年至当前日期后一年之间；
4. 参数必须为有限值；现金分红位于每10股 `[-100, 100]`，送转和配股位于每10股 `[0, 100]`，配股价位于 `[0, 100000]`；
5. 解密后候选权息记录中的无效比例不超过10%；
6. 对最多50个本地行情中可验证的除权事件，比较复权前后事件日前后跳空；复权后整体连续性不得明显恶化。

连续性检查至少需要5个可验证事件；少于5个时，只使用结构检查并显示“未完成行情连续性抽检”。检查比较事件日前后收盘价的绝对对数跳空：复权后的中位跳空不得比原始中位跳空增加超过3个百分点，90%分位跳空不得增加超过10个百分点。结构检查失败或连续性超过任一恶化阈值时，整个权息状态降级为原始价格，不允许部分证券静默混用。

界面显示具体原因，例如“有效权息记录过少”“日期分布异常”“复权连续性抽检失败”，CSV同步记录。

## 10. 匹配数量漏斗

Worker增加窗口级计数器，结果和进度协议输出：

1. 纳入扫描的证券数；
2. 实际扫描窗口数；
3. 粗筛通过窗口数；
4. 全局候选裁剪后数量；
5. 精排成功数量；
6. 精排候选中执行DTW重排的子集数量；
7. 同股去重和每股上限处理后的数量；
8. 最终展示数量。

结果页使用紧凑的横向漏斗文本和比例，不引入图表库。DTW数量显示为精排数量的括号内子集，而不是去重前的必经阶段，因为未进入DTW的精排候选仍可进入最终结果。若粗筛通过率过低或过高，给出中性参数提示，不自动修改用户设置。

## 11. 错误处理与降级

- 指数缺失：只禁用对应样本的超额统计；
- 随机池不足：报告实际配对率，不补造数据；
- 配额查询失败：只使用记录数上限；
- 缓存淘汰失败：继续当前匹配并显示缓存维护警告；
- `gbbq` 自检失败：全局降级到原始价格；
- 所有降级原因进入结果元数据、界面说明和CSV。

## 12. 测试与验收

纯函数回归测试覆盖：

- 指数日期精确对齐、节假日缺口和完全缺失；
- 日/周/月超额收益；
- 固定种子下随机基线可复现；
- 板块、日期、波动和有效性配对规则；
- recent各周期完整率和中位滞后；
- 幅度分的对称性、零波动和极端比例；
- 缓存记录数、配额、批量访问和LRU淘汰；
- 正常、损坏、记录过少和连续性恶化的 `gbbq`；
- 漏斗各阶段计数守恒。

集成测试覆盖目录扫描指数、Worker协议、页面主脚本启动、设置持久化、CSV字段、生成文件同步和本地零网络策略。

验收标准：现有测试全部通过；新增统计在指数缺失时可解释降级；固定输入产生可复现随机基线；页面能明确回答样本相对市场和随机窗口是否更好；缓存不会持续无界增长。

本次统计和评分行为变更发布时，`ALGO_VER` 与界面导出版本同步提升，确保旧缓存和CSV元数据不会与新口径混淆。
